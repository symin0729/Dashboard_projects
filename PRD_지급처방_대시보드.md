# 지급-처방 상관관계 분석 대시보드 PRD

작성일: 2026-07-08 (v1.1, pandas 기반 정밀 재검증 반영)
작성자: 민서영 (헬스케어 데이터 분석)
문서 상태: 초안 v1.1

## 1. 개요

본 대시보드는 Open Payments(제조사의 의료인 대상 지급 내역)와 Medicare Part D Prescriber(의료인별 처방 실적) 두 데이터셋을 NPI(National Provider Identifier)를 매개로 결합하여, 제조사 지급 활동과 처방 실적 간의 상관관계를 제품(product) 단위로 파악하는 것을 목적으로 한다. 주 사용자는 영업 관리자 및 리더십이며, 개별 영업 담당자 단위의 상세 관리 기능은 이번 범위에 포함하지 않는다.

본 PRD는 구현 도구를 특정 BI 솔루션으로 한정하지 않고 웹 기반 커스텀 애플리케이션을 전제로 기능 요구사항을 정의하며, 대상 데이터는 현재 제공된 2개의 정적 CSV 파일로 한정한다. 향후 데이터 소스 확장이나 실시간 연동은 범위 외로 다룬다.

v1.0 초안은 필드 구분자(콤마) 단순 분리 방식으로 데이터를 예비 검토하여 일부 수치와 데이터 품질 판단에 오류가 있었다. v1.1은 pandas를 이용한 정식 CSV 파싱으로 전체 데이터를 재검증한 결과를 반영한다.

## 2. 배경 및 데이터 현황

### 2.1 데이터 소스 개요

| 항목 | open_payments.csv | part_d_prescriber.csv |
|---|---|---|
| 레코드 수 | 36,565건 | 4,097건 |
| 고유 의료인(NPI) 수 | 7,292명 | 4,036명 |
| 지리적 범위 | CA 36,479건(99.76%), 그 외 13개 주에 86건(0.24%) 분산 | CA 4,097건(100%) |
| 제조사 | Novartis 산하 3개 법인(정규화 기준, 2.3절 참조) | 해당 없음 |
| 제품 수 | product_1 기준 22개 브랜드 | Brnd_Name 기준 3개 기본 브랜드(Entresto, Cosentyx 계열, Lucentis) |
| 총액 규모 | 지급액 합계 $1,384,132.40 | 처방 약가(Tot_Drug_Cst) 합계 $266,827,958.99 |

두 데이터셋 모두 지리적으로 캘리포니아(CA)에 압도적으로 집중되어 있다. open_payments는 CA 비중이 99.76%이고 part_d_prescriber는 100%이므로, 주(state) 단위 비교 분석은 통계적으로 의미가 없다. 이에 따라 대시보드의 지리적 프레임은 "캘리포니아 중심 분석"으로 정의하고, 나머지 13개 주의 86건은 지급 현황 참고 정보로만 별도 표기한다.

제품 범위 측면에서도 두 데이터셋의 교집합은 제한적이다. open_payments는 22개 브랜드를 포함하지만, part_d_prescriber는 Entresto와 Cosentyx 계열, Lucentis 3개 기본 브랜드만 포함한다. 이 중 Lucentis는 open_payments의 22개 제품 목록에 존재하지 않아 지급 데이터가 전혀 없으며, 따라서 지급-처방 상관관계를 산출할 수 있는 제품은 실질적으로 Entresto와 Cosentyx 2개로 한정된다. 나머지 20개 브랜드(Leqvio, Kesimpta, Xiidra, Kisqali, Mekinist, Xolair, Promacta, Tasigna, Scemblix, Tabrecta, Lutathera, Ilaris, Adakveo, Gilenya, Piqray, Zolgensma, Beovu, Mayzent, Kymriah, Vijoice)는 지급 데이터만 존재하고 대응 처방 데이터가 없다.

### 2.2 NPI 조인 커버리지

open_payments의 고유 NPI 7,292명과 part_d_prescriber의 고유 NPI 4,036명 사이의 교집합은 1,677명이다. 이는 open_payments 기준 23.0%, part_d_prescriber 기준 41.6%에 해당한다. 지급 데이터를 받은 의료인 중 약 4분의 3은 이번 처방 데이터에 대응 레코드가 없으며, 처방 데이터가 있는 의료인 중 약 58%는 지급 데이터가 없다.

이 사실은 핵심 지표 설계에 직접적인 제약으로 작용한다. NPI 기준 완전 조인(inner join)을 적용할 경우 분석 모집단이 1,677명으로 축소된다. 따라서 조인 전략은 상관관계 지표 산출용 inner join 뷰와, 지급 현황만 독립적으로 조회하는 outer 뷰를 병행 제공하는 방식으로 설계한다.

### 2.3 데이터 정합성 검토 결과

v1.0에서는 단순 콤마 분리 방식으로 데이터를 검토하여 payment_manufacturer 필드와 part_d의 주(state) 필드에 파싱 오류로 인한 데이터 손상이 있다고 판단하였다. pandas를 이용한 정식 CSV 파싱으로 재검증한 결과, 이는 데이터 손상이 아니라 필드 값 자체에 콤마가 포함된 정상 데이터였음이 확인되었다. 구체적으로 payment_manufacturer 필드의 값 중 하나는 "Novartis Gene Therapies, Inc. (fka AveXis, Inc.)"로, AveXis를 인수하여 사명을 변경한 Novartis 자회사의 정식 법인명이며 51건 모두 정상 값이다. part_d_prescriber의 주 필드 역시 전체가 정상적으로 "CA" 값을 갖고 있으며 손상 사례는 없다.

다만 실질적인 정규화가 필요한 부분은 남아 있다. payment_manufacturer 값은 대소문자 표기 차이로 "Novartis Pharmaceuticals Corporation"(36,439건)과 "NOVARTIS PHARMACEUTICALS CORPORATION"(57건)이 별도 값으로 집계되므로, 대소문자를 통일하면 해당 법인 소계는 36,496건이 된다. 이를 포함해 정규화 후 open_payments에는 Novartis Pharmaceuticals Corporation(36,496건), Novartis Gene Therapies, Inc.(fka AveXis, Inc.)(51건), Novartis Pharma AG(18건) 3개 법인이 남으며, 이는 데이터 오류가 아니라 Novartis 그룹 내 서로 다른 법인이 각자 지급을 보고한 결과로 해석된다. 대시보드에서는 이 3개 법인을 하나의 "Novartis 그룹"으로 묶어 표시하되, 법인별 세부 내역도 조회 가능하도록 한다.

또한 part_d_prescriber.csv의 GE65_Sprsn_Flag(65세 이상 수급자 수 억제 플래그)를 확인한 결과 전체 4,097건 중 1,159건('#')과 292건('*'), 합계 1,451건(35.4%)에서 CMS의 개인정보 보호 정책에 따라 수급자 수 관련 세부 수치가 억제되어 있다. 해당 레코드에 대해서는 수급자 수 기반 지표(예: 환자당 처방 비용)를 산출할 수 없다.

## 3. 목표 및 성공 기준

본 대시보드의 목표는 영업 관리자가 캘리포니아를 중심으로 제품 단위에서 지급 활동과 처방 실적의 관계를 파악하여 자원 배분 의사결정에 참고할 수 있도록 하는 것이다. 이는 다음 논리 구조를 따른다: 특정 제품에서 지급액 대비 처방액 비율이 비교 대상 제품군의 평균보다 높게 나타난다면(p), 해당 제품에 대한 영업 투자 효율이 상대적으로 높다고 잠정적으로 판단할 수 있다(q). 실제 데이터에서 이 지수를 산출한 결과 Entresto와 Cosentyx 두 제품의 결합 평균을 1.0으로 할 때 Entresto는 1.41, Cosentyx는 0.63으로 나타나, Entresto의 지급 대비 처방 효율이 Cosentyx보다 상대적으로 높다. 이는 상관관계이며 인과관계를 증명하지 않는다는 점을 화면에 명시해야 한다.

성공 기준은 다음과 같다. 첫째, 영업 관리자가 별도의 데이터 가공 없이 제품별 지급 대비 처방 지수를 3클릭 이내에 조회할 수 있어야 한다. 둘째, 고액 지급 의사 상위 목록을 통해 이상치 성격의 지급 사례를 식별할 수 있어야 한다. 셋째, 정제 로직이 재현 가능하여 매 CSV 교체 시 동일한 방법론으로 지표가 재계산되어야 한다.

## 4. 대상 사용자 및 사용 시나리오

주 사용자는 영업 관리자 및 리더십이며, 개별 NPI 단위 상세 조회보다 제품 단위 집계 뷰를 우선으로 사용한다. 대표 사용 시나리오는 다음과 같다. 영업 관리자가 특정 제품의 실적 검토를 준비하며 제품별 지급 대비 처방 지수를 비교하고, 고액 지급 의사 목록에서 처방 데이터 연계 여부를 함께 확인하여 지급 활동이 실제 처방으로 이어지고 있는지 점검한다.

접근 권한은 전체 영업팀에 공개하는 것으로 확정되었다. open_payments와 Part D Prescriber 데이터는 모두 CMS가 이미 공개한 정보이므로 공개 저장소 배포에 규제상 제약은 없다. 다만 의료인 개인의 지급 내역을 다루는 만큼, 대시보드 초기 화면에 데이터 출처와 공개 데이터 성격을 고지하는 것을 권고한다.

## 5. 데이터 요구사항

### 5.1 조인 전략

두 데이터셋은 NPI를 공통 키로 사용하여 조인한다. 상관관계 분석 뷰는 inner join 결과(1,677명, Entresto·Cosentyx 한정)를 기준으로 지급 대비 처방 지수를 산출하고, 지급 현황 뷰는 open_payments 전체(7,292명, 22개 브랜드)를 기준으로 조인 여부와 무관하게 지급 총액과 고액 지급 의사 목록을 제공한다.

### 5.2 데이터 정제 요구사항

정식 CSV 파서(따옴표로 감싼 필드 내 콤마를 인식하는 파서, 예: pandas 또는 표준 csv 모듈)를 사용해야 하며, 콤마 기준 단순 분리(split)는 필드 손상을 유발하므로 사용하지 않는다. payment_manufacturer는 대소문자를 통일하여 3개 법인으로 정규화한다. part_d_prescriber의 Brnd_Name은 펜/시린지 등 제형 표기를 제거하고 3개 기본 브랜드로 그룹화한다.

### 5.3 데이터 갱신

데이터는 정적 파일이며 수동 업데이트 방식으로 확정되었다. 사용자가 새로운 CSV 파일로 교체할 때만 대시보드 데이터가 갱신되는 구조로 설계하며, 자동 배치 파이프라인이나 실시간 연동은 이번 범위에 포함하지 않는다.

## 6. 기능 요구사항

### 6.1 핵심 지표(KPI)

제품별 지급 대비 처방 지수: 처방 약가 총합($266,827,958.99)과 지급액 총합($1,384,132.40)의 규모 차이는 약 192.8배이며, 이는 지급의 97.9%가 소액 다건의 식음료 제공이고 처방액은 인구 단위 약제비이기 때문이다. 따라서 원 비율 대신 Entresto·Cosentyx 두 제품의 결합 평균을 1.0으로 하는 표준화 지수를 사용한다. Entresto는 1.41, Cosentyx는 0.63으로 산출되어 Entresto의 지급 대비 처방 효율이 상대적으로 높다. 이 지수는 Entresto와 Cosentyx 2개 제품에 대해서만 산출 가능하며, 나머지 20개 브랜드는 지급 총액과 지급 건수만 표시한다.

고액 지급 의사(Top N) 리스트: 개별 NPI 기준 지급액 합계 상위 목록을 제공하며, 처방 데이터 연계 여부(has_prescribing_data)를 함께 표시한다. N은 기본값 30으로 하되 사용자가 조정할 수 있도록 한다.

### 6.2 필터

필터 축은 제품명(product/brand) 단위로 확정되었다. 사용자가 특정 브랜드를 선택하면 해당 제품의 지급 현황이 표시되며, Entresto 또는 Cosentyx를 선택한 경우에만 처방 연계 지표가 함께 표시된다. 그 외 20개 브랜드 선택 시 상관관계 지표 영역은 "처방 데이터 없음"으로 명시한다.

### 6.3 화면 구성

메인 화면은 제품별 지표를 카드와 막대그래프로 제공하는 요약 뷰와, 선택한 제품에 대한 고액 지급 의사 목록을 배치하는 상세 뷰 두 영역으로 구성한다. 제품 필터는 화면 상단에 고정하여 요약 뷰와 상세 뷰에 동시에 적용되도록 한다.

## 7. 비기능 요구사항

구현 형태는 웹 기반 커스텀 애플리케이션을 전제로 하며, 정적 호스팅(GitHub Pages)에서 동작하도록 사전 집계된 데이터를 정적 파일(JSON)로 내장하는 구조를 채택한다. 별도 백엔드 서버나 데이터베이스는 요구하지 않는다.

접근 권한은 전체 영업팀에 공개하며 역할 기반 접근 통제(RBAC)는 이번 범위에 포함하지 않는다. 저장소는 public으로 운영하며, 원본 데이터가 CMS 공개 데이터이므로 외부 노출에 따른 규제상 제약은 없다.

## 8. 제약사항 및 리스크

첫째, 조인 커버리지가 23~42% 수준에 그치고 상관관계 지수 산출이 가능한 제품이 Entresto와 Cosentyx 2개로 한정되므로, 지표 해석 시 대표성에 한계가 있다는 점을 사용자에게 고지해야 한다. 둘째, 지급-처방 지수는 상관관계이며 인과관계를 의미하지 않으므로, 이를 근거로 특정 의료인에 대한 지급 확대를 유도하는 방식으로 오용될 경우 Anti-Kickback Statute 등 규제 위반 소지가 있다. 이에 대한 유의 문구를 화면 내 고지할 것을 권고하며, 실제 영업 활동 지침에 대한 판단은 컴플라이언스 부서 검토를 거쳐야 한다. 셋째, GE65_Sprsn_Flag로 인해 part_d_prescriber 레코드의 35.4%에서 수급자 수 기반 지표를 산출할 수 없으므로, 해당 레코드는 처방액·처방 건수 기반 지표만 제공한다. 넷째, 두 데이터셋 모두 캘리포니아에 집중되어 있어 이번 대시보드의 분석 결과는 캘리포니아 시장에 한정된 것이며 전국 단위로 일반화할 수 없다.

## 9. 범위 외 항목

이번 PRD는 다음을 범위에서 제외한다. Novartis 외 타 제조사 데이터의 통합, 자동 데이터 갱신 파이프라인, 역할 기반 접근 통제, 개별 영업 담당자 단위 상세 관리 기능, 인과관계 추정을 위한 통계적 모델링(회귀분석 등)은 이번 범위에 포함하지 않으며 후속 단계에서 별도로 검토한다.

## 10. 배포

배포는 GitHub Pages를 통한 정적 호스팅으로 확정되었다. 저장소는 public으로 운영하며, 사전 집계된 JSON 데이터를 HTML에 내장한 단일 페이지 애플리케이션 형태로 구현한다. 원본 CSV 파일은 저장소에 포함하지 않고 집계 결과만 배포하여, 저장소 용량을 최소화하고 개별 의료인 원본 레코드의 불필요한 노출을 줄인다.
